################ INVENTORY #############################

    PURPOSE
        - Track Ingredients & Packaging
        - Link Menu Items to Ingredients
        - Update Stock on Sales / Receiving / Waste
        - Low-Stock Alerts
        - Forecasts
        - Drive Purchase Orders
    
    DATA MODEL
        - ITEMS
            - Manage Inventory Items / Locations / Batches
            - Define Recipes for Menu Items
            - Define Stock Movements (Sale / Receive / Waste / Transfer...)
            - Define Stock Levels
            - Define Suppliers
            - Purchase Orders Task

        - UNITS & CONVERSION
            - Normalize internal units
            - Conversions --> Map Packaging to Base Units
            - Validation
        
        - INTEGRATION WITH ORDERS
            - ON Order Closed --> For each sold item --> Use Recipe --> Accumulate Quantites Used
                            --> Create Stock Movements with Negative Delta Quantity
                            --> Update Stock Levels
            
        - RECEIVING / WASTE / TRANSFERS
            - Receiving --> Create Purchase Order --> Record Receive Movement --> Update Cost
            - Waste --> Record Wastes for any reason
            - Transfers --> 2 Movements in 1 Operation --> Negative A / Positive B
        
        - FORECASTING & RENDER
            - Demand --> Build Daily Consumption per Item
            - Forecast --> Moving Average or Prophet per Item
            - Reorder Suggestion --> When Stock < Reorder Threshold --> Propose Purchase Order Quantity
        
        - PERMISSIONS
            - Roles
                - Inventory Manager --> Receive / Adjust / Purchase Order
                - Staff --> View
                - Admin --> All
        
        - APIs
            - GET /api/inventory/levels?location=... --> Get Stock Levels by Location
            - POST /api/inventory/adjust --> Adjust Inventory (Admin Only)
            - POST /api/inventory/transfer --> Transfer Items
            - GET /api/inventory/items && POST --> Get & Set Items Maybe

            - GET api/recipes/:item_id && PUT --> Get or Update Recipes by Item
            - POST /api/recipes/:item_id/components
            
            - GET/POST /api/purchase-orders --> Create Purchase Orders
            - POST /api/purchase-orders/:id/receive --> 

            - GET/api/inventory/replenishment?horizon_days=... --> Get Replenishment Info

        - UI
            - Stock Overview (Location / Low-Stock Highlight / Search by SKU/Name)
            - Item Detail --> Current Levels / Recent Movements / Recipe / Suppliers
            - Receiving Screen --> Scan or Import Purchase Orders / Record Quantities / Lot/Expiry / Cost
            - Waste --> Form with Reason Codes
            - Replenishment --> Suggested Orders with Quantities and Reasons (Forecast)
            - Recipe Editor
        
        - COSTING & PRICING
            - Weighted Average --> Update (old_qty * old_cost + received_qty * unit_cost) / (old_qty + received_qty)
